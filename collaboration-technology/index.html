What to Put in Your Wiki

  • Documentation about how you did an event or campaign
    • Volunteer roles
    • Lists of supplies
    • Recruitment text
    • Schedules
    • After-event evaluation for next year's organizers

What Else?

  • How-to's for organizers
  • Meeting minutes
  • Random info you need to save like your FEIN, reservation details, insurance policy numbers, etc.
  • Any text that doesn't need lots of formatting (campaign letter drafts)

What Doesn't Go in a Wiki?

  • Things that really belong in a spreadsheet (complex financials)
  • Things that need formatting or graphic design
  • Private information about donors and volunteers

Project Management Basics

  • Make a list of to-do items
  • Give them due dates
  • Give each item an owner
  • Group together lists into a larger project

Why Not Use a Wiki?

Basecamp as an Example

We use it at CAA.

Projects

A single thing you're working on - an event, campaign, etc.

Every project contains one or more to-do lists.

To-Do Lists

One stream of work for a project, like Advertising or Website

To-Do lists contain a list of to-do items.

To-Do Items

A single task like Order posters from printer.

These have an owner and a due date.

Pervasive Discussion

Comment on projects, to-do lists, and individual items.
